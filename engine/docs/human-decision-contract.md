# Human Decision Contract

## Purpose

This document defines which questions are worth asking a human and which defaults the operator should use when the repo is otherwise complete.

The goal is to minimize unnecessary questions without forcing the operator to invent canon.

## Rule

Ask the human only when a missing answer would materially change:

- lawful planning structure
- prose truth
- build behavior
- or validation acceptance

If the question is important but not blocking, use the default below and proceed.

## Current Planning-Parity Defaults

### 1. Missing patient identity

Question shape:

"Do you want the impossible release centered on a named person, or kept concrete but unnamed for the current planning milestone?"

Default:

- keep the person concrete but unnamed
- use the stable internal identifier `released_resident_01`
- refer to the person in planning and prose by role, room relation, or chart position rather than a proper name

Reason:

This keeps the event concrete without prematurely hardening backstory while multiple route packages are being compared.

### 2. Metaphysical visibility

Question shape:

"How visibly should the deeper adjudication layer surface in the current milestone?"

Default:

- surface it lightly
- allow regime drift, judgment pressure, and ontological residue
- do not use explicit cosmology explanation or direct metaphysical exposition

Reason:

This matches the current canon and keeps the planning milestone concrete-first.

### 3. Contamination severity

Question shape:

"How harsh should contamination feel before its own route becomes executable?"

Default:

- contamination is implied but deferred from the current milestone
- allow subtle warnings, residue, and state pressure
- do not make contamination the primary route content in the current planning milestone

Reason:

The current milestone exists to prove three-route planning breadth before prose or build widening.

### 4. Retraction playability

Question shape:

"Should retraction be playable now or only preserved in planning/state during planning parity?"

Default:

- preserve retraction in canon and state logic
- do not make it a current-milestone user-facing route or ending

Reason:

This reduces scope while keeping the larger system honest.

### 5. Executable arc scope

Question shape:

"Do you want planning parity across all claimant routes, or should scope contract back to one route?"

Default:

- planning parity covers `custodian_route`, `mourner_route`, and `examiner_route`
- contamination remains canonically defined but deferred from the current milestone
- prose and build parity remain deferred until after the planning milestone is accepted

Reason:

This is already consistent with the work-instance execution milestone.

## When To Escalate Beyond Defaults

Do not use the defaults silently if:

- the human explicitly asks for named-character specificity
- a validation failure shows the defaults are structurally insufficient
- the build or graph model depends on a missing concrete answer
- the operator is asked to expand beyond the current planning milestone

## Recording Rule

When a default is used for a material decision:

- reflect it in the relevant planning artifact
- or note it in a validation report if it affected acceptance decisions

Do not treat chat memory alone as the record of that choice.
